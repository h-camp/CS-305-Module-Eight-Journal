# CS-305-Module-Eight-Journal
Briefly summarize your client, Artemis Financial, and its software requirements. Who was the client? What issue did the company want you to address?
Artemis Financial was the client for the project and they are a financial company that needed help improving the security of their software. The main goal Artemis had was to make sure their application could safely handle sensitive financial data and reduce security risks. 

What did you do well when you found your client’s software security vulnerabilities? Why is it important to code securely? What value does software security add to a company’s overall well-being?
When I was working on finding vulnerabilities, it was helpful to use the dependency-check tool to scan for issues in the project’s libraries. Coding securely is important because it helps protect user data and prevents security breaches. It also helps companies avoid bigger problems like data leaks.

Which part of the vulnerability assessment was challenging or helpful to you?
The most challenging part was understanding the report and figuring out which vulnerabilities mattered. In the beginning it was confusing, but it got easier once I started looking more closely at the details instead of just the summary. It was helpful because it taught me how to actually analyze the results.

How did you increase layers of security? In the future, what would you use to assess vulnerabilities and decide which mitigation techniques to use?
I increased security by using secure coding practices, updating dependencies and using tools to check for vulnerabilities. In the future, I will keep using tools like OWASP Dependency-Check and follow secure coding best practices to decide how to fix issues.

How did you make certain the code and software application were functional and secure? After refactoring the code, how did you check to see whether you introduced new vulnerabilities?
I tested the application after making changes and ran the dependency-check scan again. This helped confirm that the fixes worked and that I didn’t introduced any new vulnerabilities.

What resources, tools, or coding practices did you use that might be helpful in future assignments or tasks?
Some helpful tools and resources were Maven, OWASP Dependency-Check, and general secure coding practices like keeping dependencies updated. 

Employers sometimes ask for examples of work that you have successfully completed to show your skills, knowledge, and experience. What might you show future employers from this assignment?
I would show this project as an example of how I can find and fix security issues in software. It shows that I understand basic software security and how to apply it in a real project.
